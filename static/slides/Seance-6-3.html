Versions & Git
Version de texte
  • Édition savante : établir une version du texte
  • Problème du versionnage infini : mon_document_final_final_final_3.txt
  • D'une version à l'autre, des informations s'égarent
GIT
  • Gestionnaire de Versions & Parallèles de versions
  • Arborescence d'écritures
  • Repository / Répertoire des documents, des métadonnées, des historiques
En ligne / En local
  • Gestion de fichiers & projets
  • Outil complexe avec utilisation basique possible
  • Github, Gitlab, Framagit, et GitLab-HumaNum
Actions basiques
  • git add : ajout d’un fichier (nouveau ou modifié)
  • git commit : enregistrement de l’état du projet suite à une ou plusieurs modifications d’un ou plusieurs fichiers
  • git push : envoi des modifications au dépôt partagé avec d’autres personnes
Actions d'arborescence
  • git checkout : changement ou création de branches, ce qui permet d’avoir une version dupliquée du projet
  • git rebase ou git merge : fusion de plusieurs versions d’un même fichier (par exemple en réunissant plusieurs branches parallèles)
