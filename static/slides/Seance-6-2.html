Chaînes et fabriques éditoriales

Chaînes modulaires
Le Single Source Publishing
  • Publication à source unique
  • Produire plusieurs formats ou artefacts différents (de formats, de mise en page) à partir d'une seule et même source, dans le cadre d'une démarche de publication
  • Séparation contenu / mise en forme
Enjeux pour la chaîne éditoriale
  • Clarifier la chaîne qui n'est pas obligatoirement linéaire
  • Mutualiser les gestes éditoriaux
  • Limiter les coûts d'activités et la dispersion
  • Appréhender l'archivage

Modularité
  • chaîne d'édition classique : Word + InDesign
  • chaîne d'édition modulaire : un ou plusieurs modules pour une ou plusieurs fonctions

Exemples de chaînes modulaires
  • Quire : Markdown, YAML et JSON pour les données ; Hugo et PrinceXML pour la génération des formats
  • Abrüpt : Markdown, HTML, YAML ; Make, Pandoc et LaTeX pour la génération des formats

Des outils et formats qui reviennent dans le paysage du Single source publishing
  • Pandoc : logiciel de conversion en ligne de commande
  • LaTeX : langage de balisage pour la mise en forme de document vers le PDF
  • TEI : format de balisage pour l'édition savante

Exemple - Do It Yourself en Pandoc
du Docx vers le Markdown
    Exemple de commande :
  • pandoc --verbose -f doxc -t markdown Texte.docx -o Texte.md
  • cher pandoc version simple, convertis depuis (-f pour from) le DOCX vers (-t pour to) le Markdown, mon fichier de départ est "Texte.docx", et mon fichier de destination (-o pour out) sera "Texte.md"
